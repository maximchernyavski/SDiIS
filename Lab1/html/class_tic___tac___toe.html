Detailed Description

Class that contains methods and variables for realisation of Tic-Tac-Toe.

In this realisation field is presented by 1D array, for comfortable usage of which i have used such formula:

coordinate = horizontal coordinate + vertical coordinate * length of row.

I used this scheme for unifying cases for any field and lowering memory usage for algorithm on big fields.

Win combination is 3 by default and is checked by recursive algorithm.

Function that recursively checks for win in the direction.

For moving in different direction we need different signs for incrementation. We get these signs from

get_vector_signs method.

We get new coordinates based on the old ones and then recursively check for the current symbol, avoiding returning on the

visited spot.

If symbol on the spot is the current symbol, we increment delta value by 1, else we return 0, exept from case, when we

visited any spots previosly.

Parameters
xCurrent horizontal coordinate
yCurrent vertical coordinate
dirDirection of checking
deltaCombination value. Can be 3 in horizontal and vertical cases, 4 in diagonal
prev_xPrevious horizontal coordinate
prev_yPrevious vertical coordinate
